## 一、总则

### 1.1 适用范围
- **适用区间**：2026001期起全期通用
- **数据要求**：至少20期历史数据
- **预测目标**：下一期开奖生肖（12选1）

### 1.2 算法核心思想
```
十二号码分两池，八十周期轮流转。
两热两顺推四码，过渡一旧加一新。
降权解权双窗口，密集分散细甄别。
```

### 3.1 判定流程图

```
                    ┌─────────────────┐
                    │  历史数据 < 20期? │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │ 是           │ 否            │

                    │ (主池出现次数)    │
                    └────────┬────────┘     │
                             │               │
              ┌──────────────┼──────────────┐
              │ v2>v1+1      │ v1>v2+1       │ 差距≤1
              ▼              ▼               ▼
         检查V2信号      检查V1信号     检查双方信号
              │              │               │
              └──────────────┴──────────────┘
                             │
                    ┌────────▼────────┐
                    │ dominantCycle=? │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │ 有主导周期    │ 无主导周期      │
              ▼              ▼               │

              │                              │
    ┌─────────┴─────────┐                   │
    │ 有切换信号?       │ 无切换信号          │
    ├─────────┬─────────┤                   │
    │ 是      │ 否                        │
    ▼         ▼                            │
  切换到新周期 保持当前周期                  │
```

### 3.2 判定指标说明

| 指标 | 计算方式 | 用途 |
|------|---------|------|
| `v1Count` | 近20期V1主池出现次数 | 判断V1活跃度 |
| `v2Count` | 近20期V2主池出现次数 | 判断V2活跃度 |
| `cons3V1/V2` | 是否连续3期同池 | 强信号验证 |
| `cons2V1/V2` | 是否连续2期同池 | 中等信号 |
| `chainValidV1/V2` | 循环链连续性(近4期) | 链式验证 |

### 3.3 周期切换条件

满足以下**任一条件**时，切换到另一个周期：
1. 新周期主池**连续3期**开出
2. 原周期冷池**连续3期空号**

### 3.4 过渡混沌期触发条件

当以下**所有条件同时满足**时进入过渡期：
- `\|v1Count - v2Count\| <= 1`（两池持平）
- 无连续3期同池
- 无连续2期+循环链验证
- 循环链断裂

## 四、降权规则系统

### 4.1 降权触发流程

```
                    ┌─────────────────────┐
                    │ 12期内频次 >= 3次？ │
                    └──────────┬──────────┘
                               │
              ┌──────────────────┼──────────────────┐
                  │              │                     │
                  ▼              ▼                     ▼
          检查密集度      检查24期趋势          检查临界点
                  │              │                     │
          ┌───────┴───────┐     │               ┌─────┴─────┐
          │ 密集型(≤6期) │     │               │ 即将滑落?  │
          │ 分散型(>6期) │     │               │ curr=3,   │
          └───────┬───────┘     │               │ next=2    │
                  │              │               └─────┬─────┘
                  ▼              ▼                     │
          ┌───────┴───────┐     │                     │
          │24期>=5(实热)?│     │                     │
          └───────┬───────┘     │                     │
             │         │        │                     │
             ▼         ▼        ▼                     ▼
         实热+密集  假热+分散  维持≥3              豁免降权
             │         │        │                     │
             ▼         ▼        ▼                     ▼
         🔴强制降权  🟢豁免    🔴维持              🟢豁免
```

### 4.2 密集/分散判断（_isIntensivePattern）

```javascript
_isIntensivePattern(history, num, windowSize) {
    // 获取号码在窗口内的出现位置
    var positions = getNumberPositions(num);
    
    // 排序后计算跨度
    positions.sort();
    var span = 最后位置 - 最前位置;
    
    // 跨度 <= 6期 = 密集型（有连出/近间隔爆发）
    // 跨度 > 6期 = 分散型（均匀散落）
    return span <= 6;
}
```

**示例**：
```
12期内出现3次的位置：[2, 5, 11]
跨度 = 11 - 2 = 9 > 6 → 分散型 → 豁免降权 ✅

12期内出现3次的位置：[3, 4, 8]
跨度 = 8 - 3 = 5 ≤ 6 → 密集型 → 可能降权 ⚠️
```

### 4.3 双窗口交叉校验

| 12期频次 | 24期频次 | 判定结果 |
|----------|----------|---------|
| ≥3次 | ≥5次 | **实热** → 维持降权 |
| ≥3次 | <5次 | **假热脉冲** → 豁免降权 |

### 4.4 临界豁免规则（_isAtCriticalEdge）

```javascript
_isAtCriticalEdge(history, num, windowSize) {
    var positions = getNumberPositions(num);
    
    // 找最早出现的位置
    positions.sort();
    var earliestPos = positions[0];
    
    // 如果最早出现在窗口末尾(≥10)，即将滑出
    return earliestPos >= windowSize - 2;  // 10或11
}
```

**示例**：
```
12期窗口：[期1, 期2, ..., 期12]
                ↑
            最早出现(位置10)
            
→ 下期滑动后，这期将被挤出窗口
→ 频次从3降到2
→ 虽然当前是3次，但**豁免降权**
```

### 4.5 最终降权判定表

| 条件组合 | 结果 |
|---------|------|
| curr=3 + next=2 + **临界点** | 🟢 **豁免降权** |
| curr≥3 + 24实热 + **密集** | 🔴 **强制降权** |
| curr≥3 + 24假热 + **分散** | 🟢 **豁免降权** |
| next ≥ 3 | 🔴 **维持降权** |
| next === 2 + **冷却期满** | 🟡 **预备解权** |
| next === 2 + **冷却未满** | 🔴 **维持降权** |

## 五、解权规则系统

### 5.1 解权触发条件

满足以下**任一条件**进入预备解权：
1. **临界预判**：最早一期在临界点，即将滑落到2次
2. **实际滑落**：频次真实掉落至 2/1/0 次

### 5.2 强制冷却规则

```
预备解权的号码必须经过 2期无开出的静默冷却期
↓
冷却期满 → 正式解除降权
冷却未满 → 维持降权
```

### 5.3 分级解权管控

| 滑落档次 | 处理方式 | 归属池位 |
|---------|---------|---------|
| **滑落至2次** | 优先解权 | → **主推池(main)** |
| **滑落至1次** | 谨慎解权 | → **备选池(backup)** |
| **滑落至0次** | 暂缓解权 | → **静默观望**(不推荐) |

### 5.4 二次回温重锁

```
已解除降权的号码
↓
后续12期窗口再次回升 ≥ 3次
↓
立刻重新触发降权
↓
循环执行整套降权/解权规则
```

## 六、号码生成算法

### 6.1 稳定期算法（generateStableNumbers）

#### 输入
- 历史数据（按期号排序）
- 当前周期配置（V1或V2）

#### 流程

```
Step 1: 获取最近5期主池号码
        ↓
Step 2: 判断主池号码数量
        ↓
   ┌────┴────┐
   │         │
  ≥2个     1个
   │         │
   ▼         ▼
 各顺延1位  顺延2位
   │         │
   └────┬────┘
        ↓
Step 3: 从第一个顺延号开始取2个顺位号
        ↓
Step 4: 候选去重
        ↓
Step 5: filterByWeight分级过滤
        ↓
Step 6: 输出 mainNumbers(4码) + alternativeNumbers(备选)
```

#### 顺延规则详解

**情况A：最近2期都有主池号码**
```
假设V2稳定期，最近2期主池号码：[7鼠, 1马]

V2循环链：1 → 5 → 7 → 9 → 4 → 10

7的顺延 → 9（狗）
1的顺延 → 5（虎）

候选：[9, 5] + 从9开始取2顺位 → [9, 5, 4, 10]
```

**情况B：最近只有1期有主池号码**
```
假设V2稳定期，最近只有1期主池号码：[7鼠]

V2循环链：1 → 5 → 7 → 9 → 4 → 10

7的顺延 → 9（狗）
再顺延 → 4（兔）

候选：[9, 4] + 从9开始取2顺位 → [9, 4, 10, 1]
```

**情况C：最近无主池号码**
```
往前找，直到找到主池号码为止
（最多往前找5期）
```

### 6.2 过渡期算法（generateTransitionNumbers）

#### 规则
- **V1主池**：找最近1期 → 顺延1位
- **V2主池**：找最近1期 → 顺延1位
- **输出**：2码（过滤降权后）

#### 示例
```
V1主池最近：6（牛）→ 顺延 → 12（羊）
V2主池最近：7（鼠）→ 顺延 → 9（狗）

过渡期推荐：[12, 9] 或过滤后的有效号码
```

### 6.3 filterByWeight 分级过滤

#### 输入
- 候选号码列表
- 周期配置
- 降权黑名单

#### 分级处理逻辑

```
候选号码 num:
│
├── 在黑名单中？
│   └── 是 → ❌ 排除
│
└── 不在黑名单中
    │
    ├── next12频次 >= 2
    │   └── → **main池（主推）**
    │
    ├── next12频次 === 1
    │   └── → **backup池（备选）**
    │
    └── next12频次 === 0
        └── → **main池（正常参选）**
```

#### 补充机制

```
main不足4个？
│
├── 从backup提升到main
│
├── 从循环链补充（优先next12>=2或===0的）
│
└── 还不够？
    └── 从过渡区补充
```

#### 输出格式

```javascript
{
    main: [...],        // 主推4码
    backup: [...],      // 备选2码
    downWeight: [...],  // 降权黑名单
    gradeInfo: {        // 分级信息
        main: 4,
        backup: 2
    }
}
```

## 七、回测系统（runBacktest）

├── predictHistory = 历史数据[0 到 length-i-1]  （不含当期）
├── report = generateFullReport(predictHistory)  （生成预测）
├── targetItem = 历史数据[length-i-1]  （实际开奖）
├── 对比预测 vs 实际
└── 记录命中情况

## 九、执行流程（每期固定顺序）

```
1. 载入当期历史数据，生肖转标准数字
       ↓
2. 截取12期主窗口、24期辅助窗口，统计各生肖频次
       ↓
3. 判定≥3次，区分密集/分散 + 双窗口校验
       ↓
4. 生成初始降权名单（含临界豁免判断）
       ↓
5. 模拟下一期窗口滑动，做临界预判解权筛查
       ↓
6. 对预备解权生肖，校验2期冷却静默期
       ↓
7. 按2/1/0滑落档次，分级分配主推/备选/静默
       ↓
8. 调用detectCycleStage判定当前周期阶段
       ↓
9. 根据阶段调用对应算法：
   ├── V1/V2稳定期 → generateStableNumbers()
   └── 过渡混沌期 → generateTransitionNumbers()
       ↓
10. 调用filterByWeight分级过滤
       ↓
11. 输出：主推4码 + 备选2码 + 黑名单 + 周期状态
```
